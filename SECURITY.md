# Security Policy

## Reporting

If you believe you have found a security issue in this repository (e.g., leaked credentials, unsafe examples, or exploitable code snippets), please **do not** open a public issue.

Instead:
- Email the maintainer(s), or
- Use private reporting mechanisms supported by your Git hosting provider.

## Scope

This repo contains **guidance documents**. There is typically no running code, but issues can still exist:
- Secrets accidentally committed
- Unsafe copy/paste examples
- Misleading recommendations that could cause harm if deployed

## Response

We will acknowledge reports and work to remove/mitigate verified issues quickly.
